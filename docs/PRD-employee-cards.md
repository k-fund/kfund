## 1. 개요 (Overview)

### 1.1 배경
현재 회사소개 페이지(`about.html`)에는 CEO 이종현 대표 소개 섹션만 존재합니다.
회사의 신뢰도 향상과 조직 구성 소개를 위해 CEO 하단에 임직원 소개 카드를 추가합니다.

### 1.2 목표
- CEO 소개 하단에 임직원 소개 카드 영역 구현
- 관리자 대시보드에서 임직원 정보를 CRUD 관리
- 카드가 지정된 위치(좌상→우상→좌하→우하)에 순차적으로 배치

### 1.3 범위
| 포함 | 제외 |
|------|------|
| 프론트엔드 카드 UI | 임직원 로그인 시스템 |
| 관리자 CRUD 페이지 | 조직도 트리 구조 |
| Airtable 데이터 연동 | 부서별 필터링 |
| Worker API 확장 | 임직원 상세 페이지 |

## 2. 사용자 스토리 (User Stories)

### 2.1 방문자 (일반 사용자)
```
AS A 방문자
I WANT TO CEO 하단에서 임직원 정보를 확인
SO THAT 회사 구성원에 대한 신뢰감을 가질 수 있다
```

### 2.2 관리자
```
AS A 관리자
I WANT TO 대시보드에서 임직원 정보를 등록/수정/삭제
SO THAT 홈페이지에 표시되는 임직원 카드를 관리할 수 있다
```

### 3.2 카드 배치 규칙

| 순서 | 위치 | 설명 |
|------|------|------|
| 1 | 좌상 (left-top) | 첫 번째 등록 카드 |
| 2 | 우상 (right-top) | 두 번째 등록 카드 |
| 3 | 좌하 (left-bottom) | 세 번째 등록 카드 |
| 4 | 우하 (right-bottom) | 네 번째 등록 카드 |
| 5+ | 다음 행 좌상 | 5번째부터 새 행 시작 |

**배치 로직**: `order` 필드 값에 따라 2열 그리드로 자동 배치

### 3.3 카드 컴포넌트 데이터 구조

```typescript
interface EmployeeCard {
  id: string;              // Airtable Record ID
  order: number;           // 표시 순서 (1, 2, 3, 4...)
  name: string;            // 이름 (필수)
  position: string;        // 직책 (필수)
  description: string;     // 한줄 소개 (선택)
  imageUrl: string;        // 프로필 이미지 URL (선택)
  isActive: boolean;       // 공개 여부
  createdAt: string;       // 등록일
  updatedAt: string;       // 수정일
}
```

```

├── 대시보드
├── 접수내역
├── 게시판 관리

├── 방문통계
└── 설정
```

#### 기능 구성
1. **목록 화면**

2. **등록/수정 모달**
   - 이름 (필수, 텍스트)
   - 직책 (필수, 텍스트)
   - 한줄 소개 (선택, 텍스트)
   - 프로필 이미지 (선택, 파일 업로드 또는 URL)
   - 표시 순서 (숫자)
   - 공개 여부 (체크박스)

## 5. 데이터 설계

### 5.1 Airtable 테이블 구조

**테이블명**: `임직원`

| 필드명 | 타입 | 설명 | 필수 |
|--------|------|------|------|
| 이름 | Single line text | 임직원 이름 | O |
| 직책 | Single line text | 직책/직급 | O |
| 소개 | Long text | 한줄 소개 | X |
| 프로필이미지URL | URL | 프로필 사진 URL | X |
| 순서 | Number | 표시 순서 (1부터) | O |
| 공개여부 | Checkbox | 홈페이지 노출 여부 | O |
| 생성일 | Created time | 레코드 생성일 | 자동 |
| 수정일 | Last modified time | 마지막 수정일 | 자동 |

### 5.2 API 엔드포인트

**Base URL**: `https://ibn-analytics.a01027770093.workers.dev`

| Method | Endpoint | 설명 |
|--------|----------|------|
| GET | `/employees` | 임직원 목록 조회 (공개만) |
| GET | `/employees/all` | 임직원 전체 목록 (관리자) |
| POST | `/employees` | 임직원 등록 |
| PATCH | `/employees/:id` | 임직원 수정 |
| DELETE | `/employees/:id` | 임직원 삭제 |

#### API 응답 예시

**GET /employees**
```json
{
  "success": true,
  "employees": [
    {
      "id": "rec123abc",
      "이름": "홍길동",
      "직책": "이사",
      "소개": "10년 경력의 정책자금 전문가",
      "프로필이미지URL": "https://...",
      "순서": 1,
      "공개여부": true
    }
  ]
}
```

## 8. 제약사항 및 고려사항

### 8.1 기술적 제약
- 이미지 업로드: 직접 업로드가 아닌 URL 입력 방식 우선 (Airtable 파일 용량 제한)
- 순서 변경: 단순 숫자 입력 (복잡한 드래그앤드롭은 추후 개선)

### 8.2 보안 고려
- 관리자 API: 기존 인증 로직 활용 (`/admin-auth.js`)
- CORS: 기존 설정 유지

### 8.3 성능 고려
- 이미지 최적화: 권장 크기 안내 (400x400px, 200KB 이하)
- 캐싱: 프론트엔드에서 데이터 캐싱 (세션 스토리지)
